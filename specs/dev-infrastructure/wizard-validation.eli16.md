# What this PR does — in plain English

## The bug Justin hit

He typed "1" and Enter at the welcome prompt. The next step of the
wizard needs to call Codex to generate a warm intro paragraph for
the "pick a name" question — that takes 5-15 seconds. During those
seconds, the terminal showed nothing. Looked frozen.

So he pressed Enter again. That Enter got queued up in the
terminal's input buffer. When the wizard finally finished the Codex
call and asked "what's your agent's name?", it instantly read the
buffered Enter as an empty answer. The wizard's defensive default
said "if the answer is empty, call it 'agent'". And so the wizard
proceeded — "Love the name 'agent'!" — even though Justin never
typed a name.

Two real problems:
1. The user had no idea the system was working during the Codex
   call. Silence + waiting feels like a freeze.
2. The wizard accepted nothing as a name and used a default.
   Defaults are for crash recovery, not for "the user submitted
   nothing".

## What this PR fixes

### Fix 1: Visible "composing…" spinner

While Codex is generating the intro paragraph, the terminal shows a
spinning braille indicator with the word "composing…". Updates
every 100ms so the user can see the system is alive. When the
paragraph is ready, the spinner clears and the paragraph prints.
No more silent dead-air windows.

### Fix 2: Input validation per step

Each step of the wizard now has a validator. For text questions
that need a real answer (agent name, agent focus, your name), the
validator catches empty submissions and asks again with a friendly
nudge — "That looked blank, did an extra Enter slip through? Try
typing a name." For multiple-choice questions, the validator
catches garbage input and asks again with a hint about what's
acceptable.

The wizard can re-ask up to 5 times before giving up and accepting
whatever's there. That's a backstop, not a usual path — the
combination of spinner (so users don't double-Enter in the first
place) plus validator (so accidental empties get caught) should
cover normal use.

### Fix 3: Remove the silent default

The line that said "if the user's answer is empty, just use 'agent'
as the name" is gone. With the validator catching empty answers
upstream, that line was the bug. The wizard now ONLY moves on with
real input.

## Why this preserves intelligence

Justin's earlier feedback was "we need structure + intelligence,
not pure programmatic". This PR adds STRUCTURAL guards (validators,
no silent defaults) without taking anything away from the
intelligent layer. The narrative paragraphs are still generated by
Codex. The flow is still warm and conversational. The structure
just doesn't accept "nothing" as an acceptable answer to "what
should we name your agent".

## What doesn't change

- The conversation flow is the same.
- The Codex narrative per step is the same.
- The Claude wizard path is untouched.
- The agent's runtime, configuration, anything outside the wizard
  itself — untouched.

This PR is purely about making the wizard robust against the
specific UX failure Justin caught.
