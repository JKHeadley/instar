## MUST-FIX

- **B1 (adversarial + decision-completeness + gemini#5) — Six open questions block convergence.** Every §12 item is a live design decision; all six must resolve into `## Frontloaded Decisions`. Adopted resolutions: **Q1** constant 45-min dwell, no adaptive scaling in v1 (structural decision frontloaded; the numeric default carries a justified cheap-to-change-after tag — config constant, validated by the §9 soak). **Q2** reactive stays byte-identical; "reactive prefers coolest-under-ceiling" is a NAMED follow-up, not this spec; the all-hot reactive cascade is accepted explicitly with the pre-existing 5-per-10-min refresh counter named as its bound, and I6/§3.4 wording reconciled with the 120 s grace (see B10). **Q3** untagged sessions are EXCLUDED from the proactive candidate set entirely (a background optimizer must not mutate `resolveDefaultAccountId` — default-slot optimization is `POST /credentials/set-default`'s job); reactive rescue of untagged sessions is unchanged (forced) and its ledger row carries `defaultAccountChanged:true`. **Q4** v1 reads the in-memory map; ledger REQUIRED to carry the tri-state `inbound: 'reinjected'|'none'|'unknown'` (unknown = restart OR exactly-once gate dark — integration L3); CMT-1118 durable queue named as the future source (Close the Loop). **Q5** the model-swap subagent leg ships behind its own micro-flag on the model-swap config, NOT inside swapContinuity (pieces stay independently shippable; a live feature's refusal surface must not silently revert when an unrelated flag flips). **Q6** out of scope v1; the turn-boundary hook is a named follow-up with a tracking reference (cheap-to-change-after accepted: purely additive later, v1 behavior safe by construction).
- **B2 (adversarial + decision-completeness) — Breaker state does not survive restart; the spec re-creates the failure class it diagnosed (§2.4).** The ledger is named restart-safe for dwell only; an open thrash breaker (openedAt, backoff deadline, episode id) resets on boot — on exactly the restart-heavy days that motivated §2.4. FIX: breaker state is DERIVED at boot from ledger reversal rows within `reversalWindowMs` (same pattern as dwell); the episode attention item is deduped on a ledger-persisted episode id so a reconstructed episode does not re-alert.
- **B3 (adversarial + scalability S5) — A deferred intent must re-run the FULL brake pipeline at fire time, not just target revalidation.** As drafted, an intent deferred 25 min executes after its wave subsided (a pointless kill — all 8 observed waves subsided wall-free), after a reactive swap already moved the session (double kill inside the dwell window dwell exists to prevent), or after the breaker opened mid-wait. FIX: every deferral retry tick re-evaluates source-pressure ≥ threshold, dwell, reversal, breaker, all-hot, and both target bounds; an intent whose session's account changed underneath it is INVALIDATED (`intent-stale` row); a pending deferred intent is NOT exempt from a breaker that opens mid-wait.
- **B4 (scalability M1) — busy() as drafted composes the SYNCHRONOUS activity probe and stalls the event loop.** `isSessionActivelyWorking` is 2–4 blocking `execFileSync` forks (incl. a full `ps -eo` scan); spec'd frequency (deferred intents per tick + per-10 s grace re-checks across a reactive wave) = seconds of blocked loop at exactly the hot moment (mesh lease tick ~5 s, Telegram ingress on the same loop). FIX: mandate the async, coalesced probe path (`hasActiveProcessesAsync` + `tmuxExecCoalesced`, already built for this), one shared `ps` snapshot per re-check sweep; fail-direction unchanged.
- **B5 (integration M1 + lessons S3) — Cross-machine posture entirely absent.** The word "machine" does not appear in the spec; ledger/dwell/breaker are machine-local state while sessions move machines (`POST /pool/transfer`) and the same account serves multiple machines. FIX: declare everything machine-local BY DESIGN with the reason (the subscription pool is per-machine seats); state honestly that dwell does NOT follow a topic across a machine move (bounded, one-swap-wide gap; ledger-visible) and that the breaker is per-machine; declare the status fields local-scope; register cross-machine account-contention (two machines evacuating onto/off the same account) as a named tracked follow-up.
- **B6 (lessons M1) — Constitutional Traceability: no `parent-principle:` frontmatter.** FIX: `parent-principle: "No Unbounded Loops — Every Repeating Behavior Carries Its Own Brakes"` (Piece 1 is literally P19's three brakes retrofitted onto a repeating swap loop); Bounded Blast Radius + Structure beats Willpower (exact registry heading — the draft's "Structure > Willpower" is a near-miss name) as sibling principles.
- **B7 (lessons M2 + adversarial S3 + integration L4) — The gate must live at the chokepoint with a DEFAULT caller class, not in a per-caller table.** Unlisted killing callers exist today (`POST /sessions/restart-all`, manual `POST /subscription-pool/swap`, credential-repointing/escalation respawns); a table cannot bind them (the Bounded-Notification-Surface lesson shape). FIX: `SwapWorkGate` is consulted INSIDE `SessionRefresh.refreshSession`; `callerClass` defaults to `interactive-refresh` (refusal semantics) when unspecified; the brake pipeline's own chokepoint contract stated for `onQuotaPressure` (an explicit `targetAccountId` implies the caller already passed brakes; a future proactive-class caller without it gets today's reactive semantics — stated as the accepted funnel contract with rationale).
- **B8 (decision-completeness + integration S3) — §7 config block self-contradicts (`"enabled": false` vs "OMITTED → dev-agent gate").** These are different behaviors (explicit false = dark on dev too — the #1001 anti-mechanism). FIX: the key is OMITTED from shipped config (shown commented-out); §9 rung 1 names the mandatory `DEV_GATED_FEATURES` registration (+ justification line, wiring test).
- **B9 (decision-completeness) — Gate position vs the SessionRefresh rate counter unspecified.** The rate window records the attempt BEFORE the work; a deferral/refusal after that record exhausts the 5-per-10-min budget and starves the eventual legitimate swap. FIX: the busy check runs BEFORE the rate-guard record — a refused/deferred attempt consumes zero rate budget.
- **B10 (gemini M1 + adversarial L3) — Reactive grace semantics: swap at the FIRST not-busy observation; state the new-work-at-deadline outcome.** The grace is a bounded wait that proceeds at deadline; the loop must execute the swap the moment busy() first clears (not sit out the full 120 s), and if NEW work starts inside the grace it is killed at deadline WITH mitigations — acceptable and now stated (any new turn on a walled account is failing anyway). §3.4's "still swaps, immediately" corrected to "within `reactiveGraceMs` + one tick".
- **B11 (gemini M2) — Target selection order of operations underspecified.** A builder could score ALL alternates and rely on revalidation, letting drain-first scoring pick a hot "least-bad" account. FIX: normative order — FILTER to under-ceiling accounts first, THEN apply `selectAccount` scoring over that cool set only; the executed target is the survivor of filter→score→bounds.
- **B12 (decision-completeness) — Deferral-loop ownership unnamed.** FIX: `SwapWorkGate` is a STATELESS predicate (busy() only); `ProactiveSwapMonitor` owns the deferral map, `deferralAgeMs`, ceiling enforcement, and drop-at-ceiling (it owns the tick + intent lifecycle); the reactive grace loop runs as a bounded async wait inside the account-swap branch of `SessionRefresh` (never a blocking wait in a scheduler callback).
- **B13 (decision-completeness) — One authoritative ledger-row schema.** The §3.5 sketch omitted fields other sections require (`deferralAgeMs`, `killedSubagents`, `reinjectedInbound` tri-state, `subagentLeg`, transcript paths, `defaultAccountChanged`, force provenance, `failed` decisions). FIX: a single schema table naming every field, type, and which decision kinds carry it.
- **B14 (decision-completeness + lessons S2) — No three-tier test plan.** FIX: add the Testing Integrity section — unit (both sides of every brake boundary, I7's uncertainty×caller matrix), integration (route surfaces incl. the additive status fields + `session-busy`/`force`), e2e feature-is-alive through real server init, wiring-integrity for the gate's injected deps, and the P19 sustained-failure test (permanently-busy session → bounded constant-cost deferral rows → ceiling drop; permanently-failing swap → bounded retries + escalation).

## SHOULD-FIX

- **S1 (adversarial) — Reversal detector blind to different-session ping-pong.** Same-session keying misses the multi-session shape a brake-(c) regression would actually take. FIX: pair-level (any-session) reversal DETECTION feeds the breaker counter (detection-only — refusal stays same-session-keyed, preserving the legitimate 08:19 same-pair wave).
- **S2 (adversarial) — Reactive `busy-indeterminate` should resolve busy-FOR-GRACE.** The reactive arm's wait is hard-bounded (120 s), so stranding is impossible by construction; resolving indeterminate to not-busy forfeits the grace on flaky tmux exactly under load. FIX: within the reactive arm, indeterminate → wait the grace (worst case 120 s delay, never a strand); I7 restated.
- **S3 (adversarial + scalability L6) — Defer→drop→regenerate churn.** A never-idle session on a hot account cycles forever (ledger spam, pathological counters). FIX: per-(session,target-pair) re-intent backoff after `deferral-ceiling-dropped` (default = dwellMs); repeated defer rows deduped to first/last/count per intent episode.
- **S4 (scalability S2 + DC L) — Ledger read pattern.** FIX: hydrate once at boot bounded to `max(dwellMs, reversalWindowMs)` INCLUDING the newest rotated segment when the active file is younger than the window; in-memory per-session index, write-through on append; never a per-decision file scan.
- **S5 (scalability S3 + integration L2) — Rotation must name the O(1) segment helper.** `maybeRotateJsonl` is the legacy read-whole-file rewrite (marked non-conformant); FIX: `maybeRotateJsonlSegment` + cached byte counter; align numbers with the helper's house shape.
- **S6 (scalability S4) — Intra-tick target pile-on.** All `maxSwapsPerCycle` candidates can pass bounds against the SAME target on one stale snapshot — §2.2's mechanism at intra-tick scale. FIX: max 1 executed swap per target per tick.
- **S7 (security 1) — `force` is bearer-level, not operator authority.** FIX: stop attributing `force` to "the operator"; ledger row records force provenance (`force:true`, caller class); Know Your Principal wording.
- **S8 (security 2 + DC clamps) — Re-injection envelope made normative.** FIX: delimiter-collision neutralization of the quoted body; sender attribution from the stored `SenderEnvelope` (an unverified sender's text is never laundered into bare instruction position); concrete clamps — mitigation block ≤ 2000 chars, quoted inbound ≤ 1000 (ellipsized), subagent list ≤ 10 entries ("+N more").
- **S9 (lessons S1) — `failed` decision kind + foundation P19 gap.** The monitor's silent swap-failure retry (no backoff, no record) reproduces in the draft enum. FIX: `failed` added; repeated execution failure gets per-session backoff + ONE deduped attention item after N consecutive failures; §11 row added.
- **S10 (lessons S4) — Signal-vs-Authority justification stated.** The gate is blocking authority held by a low-context detector; defensible (deterministic structural-state check, same accepted class as ModelSwapService idle refusal / reaper KEEP-guards; bounded by ceiling, grace, force, recovery exemption, I7) — now argued explicitly in the spec.
- **S11 (integration S1) — I7 is not implementable through the cited binary primitive.** `isSessionActivelyWorking` swallows failure (`catch → false`) — the OPPOSITE resolution for proactive callers. FIX: the spec now names the required new tri-state probe surface (`checkSessionWorkState` returning `working|idle|indeterminate`, composing the async primitives) and drops the "invents no new detection" overclaim.
- **S12 (integration S2 + DC) — Per-key config liveness table.** The monitor snapshots knobs into readonly fields at construction; "read live" was wrong as drafted. FIX: constructor re-wired to a config getter for antiThrash knobs (live per tick); `swapContinuity.enabled` + gate wiring = restart-required; per-key table added (dynamic-MCP half-enable precedent).
- **S13 (DC) — Independence + guard posture.** swapContinuity gates refresh/model-swap paths independently of proactiveSwap; posture on `/guards` when parent dark stated (`dark-default`, never a load-bearing gap while the parent lever is off).
- **S14 (DC) — Single ledger append site.** One writer chokepoint (a `SwapLedger` module owned by the server wiring; monitor + scheduler + refresh all call it, never write the file directly).
- **S15 (DC + security 4) — `/sessions/refresh` refusal shape.** Gate check moves PRE-202: HTTP 409 `{code:'session_busy', turnInFlight, subagents:[{agentType, ageMinutes}]}`; `force:true` request field; force overrides ONLY the work gate, never the rate guard; existing respawn callers audited for the new refusal code.
- **S16 (gemini 3) — "non-baseline child process" defined by reference.** Points at the canonical baseline-filter in `SessionManager.hasActiveProcesses` (shell/tmux infrastructure exclusions) so the term is not free-floating.
- **S17 (gemini 4) — Polling justified.** The poll-based grace re-check is acknowledged as a technical constraint of the pane/proc detectors (no completion event exists to subscribe to); event-driven continuation named as a possible refinement if SubagentTracker gains completion events.
