## What Changed

Telegram poll progress now settles only after cross-machine routing acknowledges the message. On the owning machine, the durable receipt's original-event identity is authoritative even when the optional exactly-once ledger is unavailable.

## What to Tell Your User

Messages forwarded between your own machines retain one durable identity, so restarting the forwarding machine no longer repeats messages already accepted by the owner.

## Summary of New Capabilities

- Acknowledgment-coupled Telegram poll settlement.
- Restart-durable owner-side deduplication using the original platform event identity.

## Evidence

The incident reproduced on three forwarding-machine restarts: each boot re-forwarded the prior epoch's messages and the owner injected them again. Regression tests now prove the poll offset remains unadvanced while routing is unsettled and that a repeated original message ID is rejected by the durable owner receipt before restart-local fallbacks.
