# Upgrade Guide — v0.10.5

<!-- bump: patch -->

## What Changed

Supersedes v0.10.4 (which was published with a now-reverted approach). This version adds a PreToolUse hook that structurally blocks `AskUserQuestion` when the question asks for free-text input (passwords, emails, tokens, API keys, 2FA codes). Multi-choice decisions pass through normally.

When blocked, Claude receives a clear message to use plain text output instead and wait for the user's next message. The v0.10.4 approach of removing the Claude Code session entirely was reverted — the conversational setup wizard remains, now with a structural guard preventing the specific misuse pattern.

## What to Tell Your User

- **Cleaner setup experience**: "Password and credential prompts during setup are smoother now — no more confusing multi-choice menus when you just need to type something."
