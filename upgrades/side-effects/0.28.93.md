## Summary of the change

Unifies the tmux `rawInject` path in `SessionManager.ts` so every injection
(single-line and multi-line) uses bracketed-paste markers + 500 ms settle
+ Enter. Previously, single-line injections skipped the markers and the
settle, sending text + Enter back-to-back — which let Claude Code 2.1.x's
TUI eat the Enter (queued as part of the auto-detected paste buffer) and
leave the message visible-but-unsubmitted in the input box.

Adds a C0/C1 control-byte sanitizer at the input boundary, replacing
`\x1b[201~`-class byte sequences with `…` before wrapping in paste markers.
Closes a paste-exit-injection vector where a user-controlled body could
otherwise close paste mode early and execute subsequent bytes as
keystrokes.

This is a **scope cut** from the broader spec
`docs/specs/tmux-bracketed-paste-unification.md` (5-round converged,
approved by user 2026-05-12T00:45Z, then shipped against version 0.28.92).

## Decision-point inventory

  if-newline branch. Transport-layer only.

## 1. Over-block

No over-block surface. Sanitization replaces control bytes with `…`
(content modification, not rejection). The message still reaches the
agent. Tab, LF, CR are preserved.

## 2. Under-block

Pre-existing failure modes not addressed by this PR:
- Verifier flakiness for very short or whitespace-only injects — covered
  by PR #159's multi-shot verifier (3-shot escalation).
- The verifier itself depends on a brittle pane suffix match — also
  pre-existing from PR #151 and improved by PR #159.

Net change: this PR closes the root-cause delivery race for single-line
injects. PR #159's verifier remains as backstop.

## 3. Level-of-abstraction fit

Transport-layer mechanism change. `sanitizeForPaste` is a pure detector
(no authority). The unified send-keys sequence is the same proven
sequence that the multi-line path has used for months. No new gate or
authority is added.

## 4. Signal vs authority compliance

**Required reference:** [docs/signal-vs-authority.md](../../docs/signal-vs-authority.md)

- [X] No — this change has no block/allow surface. Sanitization replaces
  bytes inline; the bracketed-paste sequence is a transport mechanism.
  Both produce signals (degradation events) consumed by DegradationReporter.

## 5. Interactions

- **PR #159 verifier**: `verifyInjection` is called from `rawInject` AFTER
  the unified sequence completes, with the sanitized text as the marker.
  Behavior unchanged from PR #159 except that the marker now matches the
  bytes actually typed (since sanitization may have replaced some).
- **InputGuard**: unchanged; provenance check runs in `injectMessage`
  BEFORE `rawInject`. Sanitization happens inside `rawInject`.
- **Idle-prompt zombie detector**: `rawInject` still clears the idle
  timer at entry — unchanged.
- **No new state**: `injectSeq` map and other verifier infrastructure
  from the original spec are NOT introduced in this PR (deferred).

## 6. External surfaces

- **Other agents**: no impact.
- **Install base**: the unified path adds ~500 ms of synchronous event-
  loop blocking per single-line inject (was 0 ms; multi-line was already
  500 ms). At current load (~5–10 injects/min, single-digit concurrent
  sessions) this is well below noise threshold. HTTP request handlers
  (server.ts inject routes) inherit the +500 ms latency synchronously.
  Documented as accepted; async conversion remains a Phase-3-scale
  follow-up (spec §4.7).
- **External systems**: none directly affected. The verifier on PR #159
  emits the same DegradationReporter events as before.
- **Persistent state**: none.

New DegradationReporter event reason introduced:
- `control-byte-sanitized` (info-equivalent) — paste-marker prevention
  triggered. Bounded message preview is logged.

## 7. Rollback cost

- **Hot-fix release**: revert the single commit, ship 0.28.94 as a
  patch. No data migration, no persistent state cleanup.
- **One-file source change** (SessionManager.ts) plus the helper +
  test file. Trivial revert.
- **User visibility**: rolling back returns to the prior (occasionally-
  stuck) single-line behavior; PR #159's verifier remains in place
  either way.

## Conclusion

This PR ships the root-cause fix (bracketed-paste unification) and
defense (control-byte sanitization). Deliberately trimmed scope from
the converged spec to avoid orphaning PR #159's concurrent work.
Five review rounds on the original (broader) spec found no material
findings unaddressed at convergence; the current PR is a strict subset
plus a documented coexistence pattern with PR #159.

Ready for direct ship — no second-pass needed at this scope (second-pass
on the 0.28.92 attempt already concurred on the broader change).
